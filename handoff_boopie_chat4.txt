## Architecture notes
- Single HTML file, no dependencies except Google Fonts
- Custom `el()` / `svgEl()` / `d()` DOM helpers (no React/Vue)
- Always run Node syntax check after edits: `sed -n '139,LASTLINE p' file.html | head -n -1 > /tmp/check.js && node --check /tmp/check.js`
- Always version output files as `boopie_vN.html`
- Brace balance should always be 0
- `fmtDate(d)` uses LOCAL time; `todayKey()` uses UTC — be careful mixing them. crabStreak() and all crab logic use `todayKey()` for consistency
- Sand floor SVG: build as one HTML string with items included, inject via `div.innerHTML`. Never mix `createElementNS` + `innerHTML` + `appendChild` for SVG items.

## Key date/time gotcha
`todayKey()` uses `toISOString()` (UTC), but `fmtDate(d)` uses local time methods. For users in UTC+ timezones, these can return different dates. All crab/streak logic uses `todayKey()`. Calendar date strings come from `fmtDate()`. Don't mix them when checking if today has a crab etc.

## Key functions
- `todayActualMin()` — real time spent today (feeds pie, session timer seed)
- `todayDoneMin()` — curriculum weight done today (feeds goal check)
- `actualDoneMin(cId?)` — real time logged for completed sections
- `doneMin(cId?)` — curriculum weight of completed sections (progress bars)
- `dataScore(st)` — richness score for state object (used to pick best source on load)
- `isMeaningfulState(st)` — returns true if state has real data (guards saves)
- `isFreeTask(id)` — true if id starts with 'free_'
- `tmrSectionMin()` — current section timer in minutes
- `tmrSesMs()` — current session timer in ms

# 🐙 boopie — Project Handoff (Chat 3)

## What this is
A standalone HTML file — an adaptive music study scheduler for 5 courses. Open in any browser, no server needed. Built on the original `music_scheduler_v4.html`, now at **v56**.

## Current file
**`boopie_v56.html`** — the latest version. All changes from this chat are included.

## Hosting setup
- Hosted on **Netlify** (drag and drop to deploy)
- Connected to a **GitHub repo** called `boopie`
- **Terminal workflow** to push updates:
```
cd boopie
cp ~/Downloads/boopie_vXX.html index.html
git add .
git commit -m "vXX — description of changes"
git push
```
Netlify auto-deploys within ~30 seconds after push.

## boopie the mascot
- Egg-shaped purple octopus with 2 front arms and 6 back tentacles
- 3 moods built as SVG via `makeBoopie(mood, size)`:
  - `'idle'` — calm, arms down, gentle smile
  - `'celebrating'` — big wide eyes, huge toothy grin, arms raised, 4 back tentacles up
  - `'encouraging'` — focused brows, one arm pointing up
- Appears in: Today card, celebration overlays, Journey tab, topbar (idle, 32px), Monday/Sunday screens
- `makeBoopie()` is defined globally and returns an SVG DOM element

### Reward/incentive system
- **Tier 1 — Ocean Finds** (every 10 core sections): starfish, shells, pebbles (8 total)
- **Tier 2 — Treasures** (per chapter completed): coins, gems, keys, orbs, chest (8 total)
- **Tier 3 — Hats** (per course complete): Pirate Hat, Captain's Cap, Wizard Hat, Pearl Necklace, Golden Crown
- **Surprise tier** (random ~1/12 chance): bottle, rainbow fish, glowing chest, seahorse
- Reward celebration overlay on earn
- State: `ST.rewards` (array of `{id, earnedAt, ctx}`)
- `checkRewards()` runs on every section complete
- `maybeSurprise()` random chance on section complete

## Architecture notes (unchanged)
- Single HTML file, no dependencies except Google Fonts
- Custom `el()` / `svgEl()` / `d()` DOM helpers (no React/Vue)
- Always run Node syntax check after edits: extract `<script>` block → `node --check`
- Always version output files as `boopie_vN.html`
- Brace balance should always be 0
- `fmtDate(d)` uses LOCAL time (not UTC) to avoid timezone day-shift bugs
